Minimum Operations to Sort a Permutation

O(N) O(1) 24.2% AC
Mô tả

You are given an integer array nums of length n, where nums is a permutation of the integers from 0 to n - 1.

You may perform only the following operations:

  • Reverse the entire array.
  • Rotate Left by One: Move the first element to the end of the array, and rest elements to left by one position.

Return an integer denoting the minimum number of operations required to sort the array in increasing order. If it is not possible to sort the array using only the given operations, return -1.

 

Example 1:

Input: nums = [0,2,1]

Output: 2

Explanation:

  • Rotate Left by one: [2, 1, 0]
  • Reverse the array: [0, 1, 2]

The array becomes sorted in 2 operations, which is minimal

Example 2:

Input: nums = [1,0,2]

Output: 2

Explanation:

  • Reverse the array: [2, 0, 1]
  • Rotate Left by one: [0, 1, 2]

The array becomes sorted in 2 operations, which is minimal.

Example 3:

Input: nums = [2,0,1,3]

Output: -1

Explanation:

It is impossible to reach [2, 0, 1, 3]. Thus, the answer is -1.

 

Constraints:

  • 1 <= n == nums.length <= 105
  • 0 <= nums[i] <= n - 1
  • nums is a permutation of integers from 0 to n - 1.

Ý tưởng
<strong>1a. (Case Analysis)</strong>

The operations generate only rotations of <code>nums</code> and rotations of <code>reverse(nums)</code>. <strong>1b. (Case Analysis)</strong>

So sorting is possible if and only if <code>nums</code> is a rotation of increasing order, or a rotation of decreasing order. <strong>1c. (Case Analysis)</strong>

If <code>nums</code> is increasing cyclically and <code>nums[i] = 0</code>, then rotating left <code>i</code> times sorts the array, or we may reverse, rotate, then reverse again.

Take the minimum between these possibilities. <strong>1d. (Case Analysis)</strong>

If <code>nums</code> is decreasing cyclically, try both possibilities: reverse first, or rotate first and then reverse. Take the minimum valid cost. <strong>2a. (Breadth-First Search)</strong>

As follows from Hint 1a and 1b, each reachable array can be represented as a state. Since all reachable arrays belong to one of two families, we can compress a state into <code>(type, shift)</code>.

<code>type = 0</code>: a rotation of the original array.
<code>type = 1</code>: a rotation of the reversed array.

<code>shift</code> denotes the number of left rotations applied within that family. <strong>2b. (Breadth-First Search)</strong>

A left rotation changes <code>shift</code> to <code>(shift + 1) % n</code>.

A reverse toggles <code>type</code>, while also changing the corresponding <code>shift</code> to <code>(n - shift) % n</code>. <strong>2c. (Breadth-First Search)</strong>

Build a graph on these <code>2n</code> states using the two allowed operations, then run BFS from <code>(0, 0)</code> to the state representing the input array.
