Chương 1: Con trỏ & Quản lý bộ nhớ Lý thuyết C cơ bản

Chương 1: Con trỏ & Quản lý bộ nhớ động

Trong ngôn ngữ C, Con trỏ (Pointer) là một biến chứa địa chỉ bộ nhớ của một biến khác. Đây là công cụ cực kỳ mạnh mẽ cho phép thao tác trực tiếp trên bộ nhớ RAM vật lý. Việc hiểu rõ pointer là bước bắt buộc để tự xây dựng các cấu trúc dữ liệu phức tạp.

Bộ nhớ chương trình chia làm 2 phân vùng lớn: 1. Stack: Nơi tự động cấp phát cho các biến cục bộ khi khai báo hàm, kích thước nhỏ và tự thu hồi khi ra khỏi scope. 2. Heap: Vùng nhớ rộng lớn dùng để cấp phát động theo yêu cầu thời gian chạy (Runtime) thông qua các hàm malloc(), calloc() hoặc realloc(). Các vùng nhớ này phải được thu hồi thủ công bằng free(), nếu không sẽ dẫn đến rò rỉ bộ nhớ (Memory Leak). 

int data = 42;
int *ptr = &data; // ptr lưu địa chỉ của data (0x100)
printf("Gia tri: %d\n", *ptr); // Dereference *ptr -> 42

// Cấp phát động trên vùng nhớ Heap
int *arr = (int*) malloc(5 * sizeof(int));
if (arr == NULL) {
    printf("Cap phat bo nho that bai!\n");
    return 1;
}
free(arr); // Phải giải phóng sau khi sử dụng để tránh Leak!
Quy tắc Con trỏ an toàn: Luôn gán pointer về NULL ngay sau khi giải phóng bằng hàm free() để tránh lỗi con trỏ lơ lửng (Dangling pointer) trỏ vào vùng nhớ không hợp lệ.

Chương 2: Mảng & Mảng động (Dynamic Array)

Mảng (Array) là tập hợp các phần tử có cùng kiểu dữ liệu, nằm trong các ô nhớ liên tiếp nhau trên RAM. Ưu điểm tuyệt đối của mảng là khả năng truy cập ngẫu nhiên phần tử bằng chỉ số (Index) cực kỳ nhanh với độ phức tạp O(1). Tuy nhiên, mảng tĩnh có nhược điểm lớn là kích thước cố định, không thể thay đổi sau khi khai báo.

Mảng động (Dynamic Array) tự động khắc phục nhược điểm này. Khi mảng đầy (size == capacity), nó sẽ tự động cấp phát một vùng nhớ mới rộng gấp đôi vùng cũ, sao chép toàn bộ dữ liệu sang vùng nhớ mới và giải phóng vùng nhớ cũ. Cơ chế này đảm bảo chi phí thêm phần tử trung bình (Amortized complexity) là O(1). 

typedef struct {
    int *data;     // Con trỏ tới vùng nhớ chứa mảng
    int size;     // Số lượng phần tử hiện tại
    int capacity; // Sức chứa tối đa trước khi cần resize
} DynamicArray;

void append(DynamicArray *arr, int value) {
    if (arr->size == arr->capacity) {
        arr->capacity *= 2; // Nhân đôi sức chứa
        arr->data = (int*) realloc(arr->data, arr->capacity * sizeof(int));
    }
    arr->data[arr->size++] = value;
}
Amortized O(1): Mặc dù thao tác resize tốn O(N) để copy dữ liệu, nhưng do việc này chỉ xảy ra định kỳ rất hiếm, nên chia trung bình cho mỗi lần thêm cuối chỉ tốn đúng thời gian hằng số O(1).

Chương 3: Danh sách liên kết đơn (Singly Linked List)

Khác với mảng, Danh sách liên kết (Linked List) không lưu trữ các phần tử ở các ô nhớ liên tiếp. Thay vào đó, mỗi phần tử là một Node (Nút) được phân bố tự do trên bộ nhớ Heap. Mỗi Node bao gồm hai thành phần cơ bản: 1. Data: Giá trị thông tin cần lưu trữ. 2. Next: Một biến con trỏ chứa địa chỉ của Node tiếp theo trong danh sách. Node cuối cùng trỏ tới NULL.

Ưu điểm lớn nhất là chèn/xóa phần tử cực nhanh tại đầu danh sách với thời gian O(1) mà không cần dịch chuyển bất kỳ ô nhớ nào. Nhược điểm lớn nhất là không hỗ trợ truy cập ngẫu nhiên qua index; để tìm kiếm một phần tử, bạn phải duyệt tuần tự từ đầu (Head) tốn O(N) thời gian. 

typedef struct Node {
    int data;
    struct Node *next; // Con trỏ trỏ tới Node tiếp theo
} Node;

Node* create_node(int value) {
    Node *new_node = (Node*) malloc(sizeof(Node));
    new_node->data = value;
    new_node->next = NULL;
    return new_node;
}

Chương 4: Danh sách liên kết kép (Doubly Linked List)

Danh sách liên kết kép (Doubly Linked List) là một biến thể nâng cao của liên kết đơn. Trong đó, mỗi Node được bổ sung thêm một con trỏ thứ hai tên là prev trỏ ngược về Node đứng trước nó trong danh sách, bên cạnh con trỏ next trỏ tới Node đứng sau.

So sánh & Đánh đổi (Trade-offs): - Ưu điểm: Cho phép duyệt danh sách theo hai chiều (tiến và lùi). Thao tác xóa một Node khi đã có sẵn con trỏ trỏ tới Node đó cực kỳ nhanh chỉ tốn O(1) (trong liên kết đơn phải duyệt từ đầu để tìm Node trước nó tốn O(N)). - Nhược điểm: Tốn bộ nhớ hơn do mỗi Node phải lưu thêm 1 con trỏ prev. Thao tác chèn/xóa phức tạp hơn do phải cập nhật đồng thời nhiều con trỏ của cả các Node trước và sau. 

typedef struct DNode {
    int data;
    struct DNode *prev; // Trỏ ngược về Node trước
    struct DNode *next; // Trỏ tới Node sau
} DNode;

void insert_after(DNode* prev_node, int value) {
    if (prev_node == NULL) return;
    DNode* new_node = (DNode*) malloc(sizeof(DNode));
    new_node->data = value;
    new_node->next = prev_node->next;
    prev_node->next = new_node;
    new_node->prev = prev_node;
    if (new_node->next != NULL)
        new_node->next->prev = new_node;
}

Chương 5: Stack & Queue Cấu trúc dữ liệu trừu tượng

Chương 5: Ngăn xếp (Stack) & Hàng đợi (Queue)

Stack (Ngăn xếp) và Queue (Hàng đợi) là hai kiểu dữ liệu trừu tượng tuyến tính được sử dụng vô cùng phổ biến với quy trình truy xuất phần tử có quy luật nghiêm ngặt:

1. Stack: Hoạt động theo quy tắc LIFO (Last In, First Out - Vào sau ra trước). Thao tác chèn gọi là push(), thao tác xóa gọi là pop(). Stack giống như một ống nghiệm dựng đứng, bạn chỉ có thể thao tác ở phần đỉnh (top) của Stack. Ứng dụng: Quản lý Call Stack của hàm, chức năng Undo/Redo, thuật toán DFS.

2. Queue: Hoạt động theo quy tắc FIFO (First In, First Out - Vào trước ra trước). Thao tác chèn gọi là enqueue() ở cuối hàng (rear), thao tác xóa gọi là dequeue() ở đầu hàng (front). Giống như một hàng xếp hàng mua vé. Ứng dụng: Request Queue, hàng đợi BFS. 

// Stack cài đặt bằng mảng tĩnh cơ bản
#define MAX 100
typedef struct {
    int data[MAX];
    int top;
} Stack;

void push(Stack *s, int val) {
    if (s->top < MAX - 1) {
        s->data[++(s->top)] = val;
    }
}

Chương 6: Hàng đợi vòng Tối ưu hóa không gian lưu trữ

Chương 6: Hàng đợi vòng (Circular Queue)

Trong hàng đợi tuyến tính thông thường cài đặt bằng mảng tĩnh, sau khi một phần tử bị lấy ra (dequeue), ô nhớ ở đầu mảng sẽ bị bỏ trống và không thể tái sử dụng cho các lần chèn tiếp theo. Điều này dẫn đến sự lãng phí bộ nhớ nghiêm trọng và lỗi "Queue đầy giả".

Giải pháp Hàng đợi vòng (Circular Queue) khắc phục triệt để bằng cách kết nối ô nhớ cuối cùng quay ngược lại ô nhớ đầu tiên bằng toán tử chia lấy dư (Modulo). - Chỉ số chèn (Rear) tăng theo: rear = (rear + 1) % MAX. - Chỉ số xóa (Front) tăng theo: front = (front + 1) % MAX. - Trạng thái Queue Đầy khi: (rear + 1) % MAX == front. - Trạng thái Queue Rỗng khi: front == -1. 

typedef struct {
    int items[MAX];
    int front, rear;
} CircularQueue;

void enqueue(CircularQueue *q, int value) {
    if ((q->rear + 1) % MAX == q->front) {
        printf("Queue day!\n");
        return;
    }
    if (q->front == -1) q->front = 0;
    q->rear = (q->rear + 1) % MAX;
    q->items[q->rear] = value;
}

Chương 7: Bảng băm & Xử lý va chạm

Bảng băm (Hash Table) là cấu trúc dữ liệu ánh xạ Khóa (Keys) sang Giá trị (Values) thông qua một Hàm băm (Hash Function). Hàm băm biến đổi Key (ví dụ chuỗi ký tự) thành một chỉ số nguyên (Bucket Index) nằm trong phạm vi mảng nhằm thực hiện các phép tìm kiếm, thêm, xóa trong thời gian trung bình lý tưởng là O(1).

So sánh hai kỹ thuật xử lý Va chạm (Collision): 1. Kết nối chuỗi (Chaining): Mỗi ô mảng chứa một danh sách liên kết. Các phần tử va chạm sẽ nối tiếp nhau trên danh sách này. Dễ cài đặt, không sợ đầy bảng băm nhưng tốn thêm bộ nhớ cấp phát động cho các Node. 2. Địa chỉ mở (Open Addressing): Khi va chạm, tìm ô trống tiếp theo bằng phương pháp Dò tuyến tính (Linear Probing: (index + i) % MAX). Tiết kiệm bộ nhớ hơn nhưng dễ gây hiện tượng bó cụm (clustering) và giảm hiệu suất khi Load Factor lớn hơn 0.7. 

// Hàm băm chuỗi DJB2 nổi tiếng, phân tán dữ liệu rất đều
unsigned long hash(char *str, int capacity) {
    unsigned long hash = 5381;
    int c;
    while ((c = *str++)) {
        hash = ((hash << 5) + hash) + c; // hash * 33 + c
    }
    return hash % capacity;
}

Chương 8: Cây Heap nhị phân (Binary Heap)

Binary Heap (Cây Heap nhị phân) là một cây nhị phân hoàn chỉnh (Complete Binary Tree) được lưu trữ cực kỳ tối ưu dưới dạng một mảng tuần tự dẹt, không cần sử dụng con trỏ liên kết trái/phải. Có hai loại Heap: - Max Heap: Giá trị của nút cha luôn lớn hơn hoặc bằng các nút con của nó. Nút gốc chứa giá trị lớn nhất. - Min Heap: Giá trị của nút cha luôn nhỏ hơn hoặc bằng các nút con của nó. Nút gốc chứa giá trị nhỏ nhất.

Công thức ánh xạ vị trí nút trong mảng với chỉ số i (bắt đầu từ 0): - Con trái: 2 * i + 1. - Con phải: 2 * i + 2. - Nút cha: (i - 1) / 2. Khi chèn hoặc xóa gốc, ta thực hiện các kỹ thuật hoán đổi đổi chỗ đi lên (Heapify Up) hoặc đi xuống (Heapify Down) có độ phức tạp là O(log N). 

void heapify_up(int arr[], int i) {
    while (i > 0 && arr[i] > arr[(i - 1) / 2]) {
        int temp = arr[i];
        arr[i] = arr[(i - 1) / 2];
        arr[(i - 1) / 2] = temp; // Đổi chỗ với nút cha
        i = (i - 1) / 2;
    }
}

Chương 9: Cây tìm kiếm nhị phân Cấu trúc dữ liệu phân cấp

Chương 9: Cây tìm kiếm nhị phân (BST)

Cây tìm kiếm nhị phân (Binary Search Tree - BST) sắp xếp dữ liệu phân cấp theo quy luật: giá trị các Nút con bên Trái luôn nhỏ hơn cha, và các Nút con bên Phải luôn lớn hơn cha (Left < Parent < Right). Phép duyệt cây theo thứ tự giữa (Inorder Traversal) sẽ xuất ra một mảng đã sắp xếp tăng dần.

So sánh hai giải pháp duyệt BST (Đệ quy vs Đệ quy giả/Vòng lặp): 1. Đệ quy (Recursive): Cực kỳ ngắn gọn, dễ hiểu và dễ viết code. Tuy nhiên, nó tiêu tốn không gian bộ nhớ của Stack (Stack space complexity là O(H) với H là chiều cao cây) do phải lưu trữ nhiều khung hoạt động (stack frames) của hàm đệ quy. Có nguy cơ bị Stack Overflow nếu cây quá sâu. 2. Vòng lặp (Iterative + Stack thủ công): Phải tự tạo một Stack riêng để lưu vết, code phức tạp hơn nhưng kiểm soát tốt không gian bộ nhớ Heap và tránh được lỗi tràn ngăn xếp hệ thống. 

// Phép duyệt cây Inorder (Left-Root-Right) bằng đệ quy gọn nhẹ
void inorder(TreeNode *root) {
    if (root != NULL) {
        inorder(root->left);
        printf("%d ", root->data);
        inorder(root->right);
    }
}

Chương 10: Đồ thị & Giải thuật duyệt Mạng lưới kết nối phức tạp

Chương 10: Đồ thị & Giải thuật Duyệt (BFS / DFS)

Đồ thị (Graph) là cấu trúc biểu diễn các mối quan hệ mạng lưới giữa các đỉnh (Vertices - V) kết nối với nhau bởi các cạnh (Edges - E).

So sánh hai phương pháp Biểu diễn Đồ thị: - Ma trận kề (Adjacency Matrix): Mảng hai chiều kích thước V x V. Cho phép kiểm tra kết nối giữa 2 đỉnh trong O(1). Tuy nhiên, tốn không gian bộ nhớ lớn O(V2), không tối ưu với đồ thị thưa. - Danh sách kề (Adjacency List): Mảng chứa các danh sách liên kết. Tiết kiệm không gian bộ nhớ tối đa O(V + E), nhưng kiểm tra kết nối tốn O(V).

Hai giải thuật duyệt cốt lõi: - BFS (Breadth-First Search): Duyệt theo chiều rộng, quét từng tầng kề trước bằng cách sử dụng Hàng đợi (Queue). Độ phức tạp O(V + E). Ứng dụng: Tìm đường đi ngắn nhất. - DFS (Depth-First Search): Duyệt theo chiều sâu, đi sâu nhất có thể rồi lùi bước (backtrack) bằng Ngăn xếp (Stack / Đệ quy). Ứng dụng: Tìm chu trình, liên thông mạnh. 

// Cấu trúc DFS duyệt đệ quy dùng Call Stack hệ thống
void DFS(int v, int visited[], AdjList adj[]) {
    visited[v] = 1; // Đánh dấu đã duyệt
    printf("%d ", v);
    Node* temp = adj[v].head;
    while (temp != NULL) {
        if (!visited[temp->dest])
            DFS(temp->dest, visited, adj);
        temp = temp->next;
    }
}

Chương 11: Thuật toán sắp xếp & Độ phức tạp Big O

Sắp xếp (Sorting) là bài toán tổ chức lại các phần tử của một danh sách theo một tiêu chí xác định (tăng dần hoặc giảm dần). Có hai nhóm giải thuật sắp xếp chính: 1. Giải thuật cơ bản (O(N2)): Bubble Sort (Sắp xếp nổi bọt), Selection Sort (Sắp xếp chọn), Insertion Sort (Sắp xếp chèn). Các giải thuật này dễ cài đặt nhưng kém hiệu quả với tập dữ liệu lớn. 2. Giải thuật nâng cao (O(N log N)): Quick Sort (Sắp xếp phân chia nhanh), Merge Sort (Sắp xếp trộn), Heap Sort. Các thuật toán này tối ưu hóa tối đa số lần so sánh.

Giải thuật Độ phức tạp trung bình Trường hợp xấu nhất Bộ nhớ phụ trợ Tính ổn định
Bubble Sort O(N2) O(N2) O(1) Có
Selection Sort O(N2) O(N2) O(1) Không
Quick Sort O(N log N) O(N2) O(log N) Không 
// Code Bubble Sort: Đổi chỗ các cặp phần tử liền kề nếu sai thứ tự
void bubble_sort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j+1]) {
                int temp = arr[j];
                arr[j] = arr[j+1];
                arr[j+1] = temp; // Hoán đổi (Swap)
            }
        }
    }
}
